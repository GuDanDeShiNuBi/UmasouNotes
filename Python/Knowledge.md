* 在 Python 中，* 和 ** 是解包操作符，它们的作用是将 “集合型参数” 拆分成 “独立的参数” 传递给函数，而不是将整个集合作为一个参数传入。
  * *args：将可迭代对象（如元组、列表）拆成多个位置参数（按顺序依次传递）
    例如 args = (1,2,3)，则 *args 等价于 1,2,3（三个独立的位置参数）
  * **kw：将字典拆成多个关键字参数（按 key=value 形式传递）。
    例如 kw = {'d':99, 'x':'#'}，则 **kw 等价于 d=99, x='#'（两个独立的关键字参数）
* list（列表）是可变的：创建后可以修改、添加、删除元素
* 元组的 “不可变性” 指的是元组自身的元素不能被修改（无法给元组的某个索引重新赋值），但这并不影响变量可以引用新的值。
* x for x in range(10) 是一个生成器表达式，可以带（）,也可以不带。生成器本质上是一种特殊的迭代器
  * 迭代器的核心定义
    迭代器是一个可以被 next() 函数调用并返回下一个值的对象，它必须满足：
    有 __iter__() 方法：返回自身（使迭代器可在 for 循环中使用）。
    有 __next__() 方法：返回下一个元素，当没有元素时抛出 StopIteration 异常。
    生成器表达式创建的生成器对象天然实现了这两个方法，因此符合迭代器的定义。
    生成器表达式的 “惰性计算” 特性
    生成器表达式不会像列表推导式（[x for x in range(10)]）那样一次性生成所有元素并存储在内存中，而是按需生成元素：
    当调用 next() 函数时，才会计算并返回下一个元素；
    元素生成后不会被保存，用完即丢弃（节省内存）。
    这种 “惰性计算” 是迭代器的典型行为（迭代器本质就是 “流式” 产生数据的对象）。
* list、dict、str等数据类型不是Iterator（迭代器）,是Iterable（可迭代对象）。
  * 没有 __next__() 方法，迭代器的核心能力是通过 __next__() 方法 “主动产生下一个元素”
  * 不支持 “惰性计算”，list 和 dict 在创建时会一次性存储所有元素（占用内存），而迭代器的特点是 “按需生成元素”（不提前存储，节省内存）。
  * list 和 dict 的 __iter__() 方法返回一个新的迭代器（而非自身），而迭代器的 __iter__() 方法返回自身（确保迭代器可被 for 循环重复使用）。
* list 支持 for 循环，是因为它能提供迭代器；for 循环中调用 next() 的是这个 “临时迭代器”，而非 list 本身。list 是可迭代对象（提供迭代器）
* map()函数接收两个参数，一个是函数，一个是Iterable，map将传入的函数依次作用到序列的每个元素，并把结果作为新的Iterator返回。map 对象不会提前计算并存储所有结果，而是在被迭代时 “按需计算”.
